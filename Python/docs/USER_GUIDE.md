# Руководство пользователя

## Назначение

Приложение "Путеизмерительная тележка" предназначено для настройки подключения к микроконтроллеру, калибровки датчиков и сохранения данных проездов в CSV-файлы.

Окно приложения состоит из нескольких основных зон:

- параметры сохранения данных;
- выбор COM-порта;
- индикаторы текущих показаний датчиков;
- кнопки управления процедурой измерений;
- журнал информационных сообщений;
- номер текущего проезда.

## Подготовка к работе

Перед запуском измерений убедитесь, что:

- микроконтроллер подключён к компьютеру;
- COM-порт устройства доступен в системе;
- выбрана директория, куда можно записывать файлы;
- задано шаблонное название файлов.

## Настройка сохранения и подключения

1. В поле `Директория сохранения данных` укажите папку для результатов.
2. В поле `Шаблонное название файлов` укажите базовое имя файлов.
3. В списке `Выбранный com порт` выберите порт микроконтроллера.
4. Если нужного порта нет в списке, нажмите кнопку обновления рядом со списком COM-портов.
5. Нажмите `Применить`.

После применения настроек приложение:

- блокирует изменение параметров сохранения и COM-порта;
- запускает сессию связи с микроконтроллером;
- ожидает рукопожатие с МК;
- выводит результат в журнал информационных сообщений.

Если рукопожатие выполнено успешно, можно запускать калибровку.

## Калибровка датчиков

1. Нажмите `Начать калибровку датчиков`.
2. Дождитесь завершения процедуры.
3. После завершения приложение покажет информационное окно `Калибровка завершена`.
4. Подтвердите сообщение нажатием `OK`.

После калибровки приложение переходит к этапу набора статического буфера.

Повторная калибровка доступна после завершения текущего проезда. При повторной калибровке приложение предупреждает, что текущий цикл подготовки к измерениям будет сброшен и статический буфер нужно будет набрать заново.

## Набор статического буфера

Перед каждым измерением необходимо заново набрать статический буфер.

1. Убедитесь, что тележка находится в состоянии, подходящем для набора статического буфера.
2. Нажмите `Начать сбор статического буфера`.
3. Дождитесь завершения процедуры.
4. После завершения приложение покажет информационное окно `Статический буфер набран`.
5. Подтвердите сообщение нажатием `OK`.

Во время набора статического буфера приложение сохраняет данные в файл:

```text
{шаблонное_название}_static_init_{номер_проезда}.csv
```

Например:

```text
telega_2026-05-19_static_init_1.csv
```

Повторный запуск набора статического буфера во время активной процедуры блокируется.

## Выполнение проезда

После завершения набора статического буфера приложение готово к измерению.

1. Нажмите `Начать проезд`.
2. Выполните проезд тележки.
3. Для завершения измерения нажмите `Завершить проезд`.

Во время проезда приложение сохраняет данные в файл:

```text
{шаблонное_название}_measuring_{номер_проезда}.csv
```

Например:

```text
telega_2026-05-19_measuring_1.csv
```

После завершения проезда:

- текущий CSV-файл закрывается;
- номер текущего проезда увеличивается на 1;
- приложение возвращается к этапу набора статического буфера.

## Последовательность работы

Типовой рабочий цикл выглядит так:

1. Задать директорию сохранения, шаблон файла и COM-порт.
2. Нажать `Применить`.
3. Дождаться успешного рукопожатия с МК.
4. Нажать `Начать калибровку датчиков`.
5. Дождаться сообщения о завершении калибровки.
6. Нажать `Начать сбор статического буфера`.
7. Дождаться сообщения о завершении набора статического буфера.
8. Нажать `Начать проезд`.
9. После окончания проезда нажать `Завершить проезд`.
10. Для следующего проезда снова набрать статический буфер.

## Индикаторы и журнал сообщений

В зоне показаний датчиков отображаются текущие значения:

- `Acc_X`, `Acc_Y`, `Acc_Z`;
- `Gyro_X`, `Gyro_Y`, `Gyro_Z`;
- `Код ДПП`.

Справа расположен журнал `Информационные сообщения`. В него выводятся важные события:

- применение настроек;
- ожидание и результат рукопожатия;
- запуск и завершение калибровки;
- запуск и завершение набора статического буфера;
- запуск и завершение проезда;
- предупреждения и ошибки.

Номер текущего проезда отображается в верхней части окна. Он изменяется автоматически после завершения проезда.

## Ограничения действий

Приложение контролирует порядок работы и предупреждает пользователя, если действие сейчас недоступно.

Нельзя:

- запускать калибровку до применения настроек;
- начинать набор статического буфера до успешной калибровки;
- начинать проезд до завершения набора статического буфера;
- повторно запускать набор статического буфера во время активного набора;
- запускать новый проезд, пока предыдущий не завершён;
- менять параметры сохранения и COM-порт после начала рабочей сессии.

## Завершение работы

Для завершения работы закройте окно приложения стандартной кнопкой закрытия.

При закрытии приложение:

- останавливает сессию связи с микроконтроллером;
- завершает чтение очередей данных;
- закрывает открытый CSV-файл, если он был открыт;
- освобождает ресурсы COM-сессии.

Если проезд был активен, перед закрытием рекомендуется сначала нажать `Завершить проезд`, чтобы текущий файл измерений был закрыт штатно.

## Возможные ошибки

Если приложение сообщает об ошибке задания параметров, проверьте:

- существует ли директория сохранения или можно ли её создать;
- доступна ли директория для записи;
- задано ли корректное шаблонное название файлов;
- выбран ли COM-порт.

Если рукопожатие с МК не выполнено:

- проверьте физическое подключение устройства;
- убедитесь, что выбран правильный COM-порт;
- проверьте, не занят ли порт другой программой;
- повторно примените настройки после устранения причины.

Если возникла ошибка записи CSV:

- проверьте свободное место на диске;
- убедитесь, что файлы результатов не открыты в другой программе с блокировкой записи;
- проверьте права записи в директорию сохранения.
